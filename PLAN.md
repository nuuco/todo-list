# 바닐라 JavaScript Todo 앱 개발 계획

## 구현 방향
- 기술 선택: 바닐라 JavaScript + HTML + CSS
- 다크 모드: 사용자 수동 토글 방식(버튼)으로 적용
- 저장 방식: `localStorage` 기반 영구 저장
- 상태 관리: 단일 `state` 객체 + 렌더 함수 재실행으로 즉각 UI 반영

## 파일 구성(신규 생성)
- index.html
- styles.css
- script.js
- README.md
- AGENTS.md

## 기능 개발 단계
1. 기본 레이아웃 및 반응형 구조 작성
- 입력 폼, Todo 리스트, 필터 버튼(전체/진행/완료), 남은 개수, 다크모드 토글 UI 구성
- 모바일 우선 CSS로 시작 후 태블릿 브레이크포인트 확장

2. Todo 데이터 모델 및 저장소 연동
- Todo 항목 구조: `id`, `text`, `completed`, `createdAt`
- 앱 초기화 시 `localStorage` 로드, 변경 시 저장

3. CRUD 및 유효성 처리
- 추가(Create): 빈값/공백 입력 방지 및 경고 문구 표시
- 조회(Read): 필터 조건에 맞는 목록 렌더
- 수정(Update): 완료/미완료 토글 시 즉시 화면 갱신
- 삭제(Delete): 개별 삭제 기능 반영

4. 필터 및 실시간 카운트
- 필터 상태(`all`, `active`, `completed`)를 상태값으로 관리
- 렌더 시 남은 할 일 개수를 실시간 계산/표시

5. 다크 모드 토글
- 토글 클릭으로 라이트/다크 테마 클래스 전환
- 선택한 테마를 `localStorage`에 저장해 재방문 시 복원

6. 문서화
- `README.md`에 아래 섹션 작성
  - AI 활용 프롬프트 기록
  - 주요 기능 요약 문서
  - 기술 스택
  - 개발 회고록(제목만 생성, 본문 비워두기)
- `AGENTS.md`에 개발 단계별 이력 한 줄 형식으로 기록

## 검증 계획
- 입력 공백/빈값 차단 동작 확인
- CRUD 전 과정에서 화면 즉시 반영 확인
- 새로고침 후 데이터/테마 유지 확인
- 전체/진행/완료 필터 정확성 확인
- 모바일/태블릿 해상도에서 레이아웃 깨짐 여부 확인
