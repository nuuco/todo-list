# 사용한 프롬프트 로그

- 프롬프트:
```
자바스크립트를 이용해서 Todo 앱을 만들거야. 아래 요구사항을 확인해서 개발 계획을 작성해줘. 불분명하거나 의사결정이 필요한 부분 있으면 나에게 질문해.

## 요구사항
- 반응형 웹 (모바일, 태블렛 대응)
- Todo CRUD
- 빈값 방지
- 리스트 표시
- 완료 / 미완료 상태 변경
- 상태 변경시 화면 즉각 갱신
- 다크 모드 적용
- LocalStorage로 데이터 영구 저장
- 상태 필터링(전체, 진행, 완료)
- 남은 할일개수 실시간 표시

## 추가 작업
- README 문서에 'AI 활용 프롬프트 기록', '주요 기능 요약 문서', '기술 스택' 및 '개발 회고록'이 작성되어야함.
- 개발 회고록은 제목만 만들고 작성하지 말것 (직접 작성 예정)
```
> 의도/반영: 요구사항을 구현 단계로 분해하고, 확인이 필요한 기술 선택/다크모드 방식을 먼저 질문한 뒤 계획 수립

- 프롬프트:
```
첨부된 계획 문서를 기준으로 그대로 구현해줘.
계획 파일 자체는 수정하지 말고, 이미 생성된 To-do를 순서대로 in_progress로 바꾸면서 끝까지 완료해줘.
```
> 의도/반영: 계획 파일은 유지하고 구현만 진행, To-do 상태를 순차 갱신하며 기능 전체 구현

- 프롬프트:
```
스크립트 파일 이름은 `script.js`로 바꾸고,
계획 문서는 루트에 `PLAN.md`로 저장해줘.
```
> 의도/반영: `app.js`를 `script.js`로 변경하고 `PLAN.md`를 프로젝트 루트에 생성

- 프롬프트:
```
사용한 프롬프트 기록을 별도 파일로 정리해줘.
README에는 프롬프트 로그와 계획 문서 링크를 연결해줘.
```
> 의도/반영: 프롬프트 로그를 별도 파일로 분리하고 README에서 로그 문서/플랜 문서 링크 제공

- 프롬프트:
```
PROMPT_LOG의 가독성을 높여줘.
번호는 제거하고 불릿 형식으로 바꾸고,
의도/반영은 인용구 형태로 표기해줘.
```
> 의도/반영: 로그 형식을 불릿+인용구 구조로 개편해 가독성 개선

- 프롬프트:
```
각 프롬프트는 복사하기 쉽게 코드블록으로 감싸서 정리해줘.
```
> 의도/반영: 각 프롬프트를 코드블럭으로 감싸 재사용성 향상

- 프롬프트:
```
참고 이미지처럼 단정하고 미니멀한 분위기로 스타일을 바꿔줘.
```
> 의도/반영: 미니멀 체크리스트 톤으로 UI 전면 리디자인

- 프롬프트:
```
테마 전환 버튼은 텍스트 대신 심플한 아이콘 형태로 바꿔줄 수 있을까?
```
> 의도/반영: 테마 토글을 텍스트 버튼에서 아이콘 버튼으로 전환

- 프롬프트:
```
제목과 할 일 본문만 명조 계열을 유지하고, 나머지는 고딕으로 적용해줘.
추가/삭제 버튼은 텍스트 대신 아이콘으로 처리해줘.
```
> 의도/반영: 타이포 체계 분리(제목/할일만 명조) 및 액션 버튼 아이콘화

- 프롬프트:
```
라이트 모드 배경을 조금 더 밝게 조정해줘.
입력 placeholder는 명조 느낌으로 맞춰주고,
삭제 버튼은 배경 없이 빨간 아이콘만 보이게 바꿔줘.
```
> 의도/반영: 색상/타이포 미세 조정 및 삭제 버튼 시각 스타일 개선

- 프롬프트:
```
테마 버튼은 화면 우측 상단으로 이동해줘.
제목-입력 사이 간격은 줄이고,
입력-필터 간격은 줄이되 필터 아래 간격은 더 넓혀줘.
```
> 의도/반영: 헤더/섹션 레이아웃 간격을 화면 감성에 맞게 조정

- 프롬프트:
```
추가/삭제는 SVG 대신 태그 기반 기호로 바꿔줘.
버튼 기호는 아래로 약 2px 내리고,
테마 아이콘은 반대로 2px 올려서 중심 맞춰줘.
```
> 의도/반영: 액션 버튼을 문자 기호(`+`, `×`) 기반으로 전환하고 수직 정렬 보정

- 프롬프트:
```
삭제가 간헐적으로 동작하지 않으니 먼저 원인을 찾아주고 바로 수정해줘.
```
> 의도/반영: 이벤트 위임 조건을 `closest`로 교체해 삭제 동작 버그 해결

- 프롬프트:
```
더 이상 쓰지 않는 삭제/추가 아이콘 파일은 정리해줘.
```
> 의도/반영: 미사용 SVG 자산 정리

- 프롬프트:
```
삭제 버튼도 항목 상단 정렬로 맞춰줘.
체크박스뿐 아니라 텍스트 클릭으로도 완료/미완료가 전환되게 해줘.
입력을 다시 시작하면 안내 메시지는 즉시 사라지게 처리해줘.
```
> 의도/반영: 상호작용 UX 개선(정렬, 텍스트 토글, 메시지 초기화)

- 프롬프트:
```
입력창 아래에 `현재 글자수/최대 글자수`를 표시해줘.
표시는 밑줄 아래에 두고, 안내 메시지와 같은 라인에 배치해줘.
```
> 의도/반영: 입력 하단 메타 라인(안내 메시지 + 글자수 카운터) 구성

- 프롬프트:
```
최종 코드 점검 후 비효율/불필요 코드를 정리해줘.
```
> 의도/반영: 중복 로직 제거, 공통 함수화, 불필요 분기 정리 중심의 리팩터링 수행

- 프롬프트:
```
완료 시 콘페티 같은 애니메이션을 넣고 싶어.
모든 할 일 완료일 때는 더 큰 축하 효과를 적용해줘.
```
> 의도/반영: 단일 완료/전체 완료를 분리한 단계형 연출(콘페티, 배지, 사운드) 추가

- 프롬프트:
```
할 일이 많아졌을 때 페이지 전체가 아니라 리스트 영역 안에서만 스크롤되게 하고,
스크롤바도 화면 분위기에 맞게 스타일링해줘.
```
> 의도/반영: 리스트 전용 스크롤 구조로 전환하고 커스텀 스크롤바 적용

- 프롬프트:
```
텍스트 클릭으로 완료 토글하던 동작은 제거하고,
텍스트 클릭 시에는 할 일 내용을 수정할 수 있게 바꿔줘.
```
> 의도/반영: 텍스트 클릭 인라인 편집 모드 도입, 저장/취소 키 동작과 포커스 아웃 처리 구현

- 프롬프트:
```
편집 중인 상태가 더 잘 보이게 UI를 보강해줘.
편집 행 여백, 라벨 위치/색상, 입력 영역 스타일을 미세 조정해줘.
```
> 의도/반영: 편집행 인지성 향상(상하 여백 확대, "편집 중" 라벨 위치/톤 조정, 배경/보더 정책 정리)
